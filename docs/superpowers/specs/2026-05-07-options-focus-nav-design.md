# Options 页面专注模式导航同步与磁吸切页设计

## 背景
当前 `options/` 页面采用外层壳 + `iframe#content-frame` 的结构：左侧 sidebar 负责页面切换，右侧悬浮点导航（`dot-nav`）只在点击切页时同步高亮。专注模式已经存在，但它只负责隐藏 sidebar，没有把“当前页高亮”和“滚动到边界继续滚动自动切页”串起来。

## 目标
1. 进入专注模式后，右侧导航始终高亮当前 iframe 内显示的页面。
2. 在专注模式下，用户滚动到页面顶/底后继续滚动时，自动切换到上一页/下一页。
3. 切页支持循环：最后一页继续向下切到第一页，第一页继续向上切到最后一页。
4. 切页后新页面的滚动位置要吸附到边界：下一页滚到顶部，上一页滚到底部。
5. 内部可滚动区域优先：例如 textarea、JSON 编辑框、书签树等区域还能继续滚动时，不触发整页切换。

## 范围
### 会修改
- `options/options.js`
- `options/options.html`（如需挂载公共脚本或增加标记）
- `options/options.css`（如需新增专注态或导航态样式）
- 各 options 子页面脚本（`options/platform/platform.js`、`options/api/api.js`、`options/storage/storage.js`、`options/menu/menu.js`、`options/notes/notes.js`、`options/ocr/main.js`、`options/countdown/countdown.js`、`options/prompts_editor/prompts_editor.js`、`options/local_cmd/*.js` 等）

### 不会修改
- 主业务模块、popup、content script、后台消息流
- 非 options 的页面布局与导航逻辑

## 交互设计
### 专注模式高亮同步
- `options/options.js` 继续作为唯一的页面切换控制器。
- 每次 iframe src 改变时，sidebar 和 dot-nav 都更新 active 状态。
- 当子页面触发磁吸切页时，父页面复用同一套 `navigateToPage()` / `updateNavRingActive()` 逻辑，避免出现两个导航系统状态不一致。

### 磁吸切页触发
- 仅在专注模式生效。
- 在 iframe 内监听 `wheel` 和必要的滚动状态变化。
- 先判断事件目标链路上是否存在“还能继续滚”的内部滚动容器。
- 只有当内部滚动已到边界，并且页面本身也已到边界时，才把 wheel delta 累积到阈值。
- 累积超过阈值后，向父页面发送翻页请求：`next` 或 `prev`。

### 切页边界规则
- 页面顺序沿用现有 `NAV_ITEMS` 顺序。
- 到最后一页继续向下 -> 第一页。
- 到第一页继续向上 -> 最后一页。
- 翻页完成后：
  - `next` -> 新页面滚到顶部
  - `prev` -> 新页面滚到底部

## 组件拆分
### 1. 父页面导航控制器
职责：
- 读取并恢复 selected tab / 专注模式状态
- 处理 sidebar、dot-nav 的 active 状态
- 处理 iframe src 切换
- 接收来自子页面的翻页请求并执行循环切页
- 在新页面加载完成后发送“滚到边界”的指令

依赖：
- `NAV_ITEMS`
- `STORAGE_KEYS`
- `content-frame`

### 2. 子页面滚动代理
职责：
- 仅在专注模式启用时工作
- 识别事件是否来自可滚动内部区域
- 区分“内部区域还能滚”与“页面已经到边界”
- 累积边界 wheel delta 并上报翻页请求
- 接收父页面的滚动吸附指令并执行

依赖：
- `window.parent.postMessage`
- `postMessage` 来源校验（仅处理本扩展同源父页面）
- 当前页面的滚动状态

### 3. 可复用滚动判断工具
职责：
- 判断某个元素是否可滚动
- 判断某个元素在指定方向上是否还能继续滚动
- 从事件 target 向上寻找最近可滚动祖先

这部分逻辑需要保持小而明确，避免每个页面都写一遍不同版本的边界判断。

## 数据流
1. 用户进入专注模式。
2. iframe 内页面挂载滚动代理逻辑。
3. 用户在页面上滚动。
4. 若内部滚动区还能继续滚动，则仅执行内部滚动。
5. 若页面到顶/到底，继续滚轮会累积到阈值。
6. 达到阈值后，子页面向父页面发送 `focusScrollNavigate` 消息。
7. 父页面计算下一页/上一页，更新 iframe src，并同步 sidebar / dot-nav active。
8. 新页面加载完毕后，父页面要求它滚到顶部或底部。

## 错误处理
- 如果子页面未收到专注模式状态，默认不启用磁吸逻辑，避免普通模式受影响。
- 如果父页面收到未知 message action，直接忽略。
- 如果切页目标不存在，回退到当前页，不改变状态。
- 如果新页面尚未完成加载，滚动吸附指令延后到 load 完成后再执行。
- 如果内部滚动区判断失败，宁可不触发切页，也不要误切页。

## 验证方式
### 需要验证的场景
1. 专注模式开启后，dot-nav 高亮与当前 iframe 页面一致。
2. 手动点击 sidebar / dot-nav 切页后，高亮同步正确。
3. 在页面底部继续向下滚动时，自动切到下一页。
4. 在页面顶部继续向上滚动时，自动切到上一页。
5. 最后一页继续向下会循环回第一页；第一页继续向上会循环回最后一页。
6. JSON 编辑框、书签树、textarea 等内部滚动区在还能滚动时不会触发切页。
7. 切页后滚动位置符合吸附规则：next 到顶部，prev 到底部。

### 重点检查
- 不影响普通模式滚动
- 不影响 iframe 内原有交互
- 不影响消息驱动的 countdown 跳转逻辑
- 不引入重复监听或重复注册

## 实现约束
- 保持现有 iframe 架构，不改成单页路由。
- 尽量把滚动判断做成可复用的公共函数，减少各子页重复代码。
- 不把翻页逻辑散落到多个页面里，最终切页只能由父页面统一执行。
- 专注模式逻辑只做 UI 级别行为，不改变任何存储数据结构。
